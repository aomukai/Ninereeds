# Mommy Says Machine

Mommy Says Machine is the active training loop for Ninereeds.
For executable step order, use `runbook.md`; this file defines the system boundary.

Older docs described MSM as a diagnostic intervention around traditional corpus training.
That role is deprecated. MSM now means scripted teaching, local execution, grading,
reporting, repair, and optional micro-updates.

---

## Purpose

MSM answers:

1. What does Ninereeds currently do when asked about this word?
2. Which axes are stable, weak, wrong, or off-topic?
3. Which correction shape helps?
4. Which proposed turns are safe enough for orchestrator review?
5. Should the executor continue locally, or should the orchestrator decide strategy?

---

## Machine Boundary

MSM is a pipeline, not one model.

- The orchestrator owns strategy, queue order, limits, and update decisions.
- The executor writes scripts, grades logs, writes reports, and proposes trainable turns.
- The trainer executes fixed scripts and writes clean logs.
- Hermes reports status and pings the user when needed.

This separation is intentional. The worker that runs prompts does not grade them, and the
worker that grades logs does not own the long-range research plan.

The trainer may be a Python script. It does not need to be a small language model.

---

## Session Structure

Each session targets one word/card.

Inputs:

- parent checkpoint
- concept card
- orchestrator policy/plan
- script generated by the executor

Outputs:

- raw chat log
- fixed report card
- turn-level grades
- failed-turn diagnosis
- optional proposed training turns

---

## Scripted Teaching Shape

A normal contrast item looks like:

```text
[user]Is a dog a cat?
[Ninereeds]A dog is an animal. A cat is an animal. A dog is a cat.
[teacher]A dog is not a cat.
[Ninereeds]A dog is not a cat.
```

The trainer records this faithfully. The executor later grades:

```text
original answer: incorrect, on-topic
after correction: correct
```

Every scripted item is graded independently.

---

## Mommy Says Correction Rule

Corrections:

- restate the correct form
- avoid praise and judgment
- are short
- target one failure mode at a time
- use known support vocabulary when possible
- are retested immediately when the script requests a post-correction answer

Do not train from the wrong Ninereeds answer.

---

## Report-First Design

The executor must fill `report_card.json` after every session. The orchestrator should be
able to compare sessions without parsing inconsistent prose.

Required schema: `training/pipeline/session_report_schema.md`.

---

## Executor Auto-Advance Boundary

The executor may continue to the next local script only while the report evidence remains
inside the active policy:

- at least one scripted item has a correct original answer or correct post-correction answer
- no answer is off-topic
- malformed/repetition thresholds are not exceeded
- retry/script limits are not exhausted

The executor escalates to the orchestrator when:

- no scripted item receives a correct answer
- at least one answer is off-topic
- repeated failures exceed policy
- protected anchors fail
- an update/promotion decision is ready
- grading confidence is too low

---

## Closed Loop

MSM can update weights, but only through orchestrator-approved turns:

1. Raw session is logged.
2. The executor grades turns.
3. The executor proposes grading-validated corrections/expected answers.
4. The orchestrator approves selected turns into an update buffer.
5. A buffered micro-update creates an update-candidate checkpoint from approved turns.
6. Target checks and protected anchors decide acceptance.

Raw logs are never ingested directly. Executor-proposed turns are not ingested until the
orchestrator copies approved records into `approved_training.jsonl`.

---

## Rollback-Replay

If a session reveals harmful confusion, the default repair is rollback-replay:

1. Return to the pre-session checkpoint.
2. Generate a repair replay script for the failure mode.
3. Execute and report the replay.
4. Accept only if the repair holds and protected anchors pass.

This is the standard response to failures such as:

- `a cat is a dog`
- `a tree is a mammal`
- identity/name confusion
- unknown-boundary failure
- topic drift that becomes stable

---

## Cold Start

MSM begins from random weights. Early logs may contain byte noise, letter fragments,
malformed words, and nonsensical sentences. Cold-start reporting still uses stable
artifacts, but each phase has its own frontload, evaluation, and gate criteria.

---

## Human Attention

The orchestrator or Hermes writes a sentinel file when automation cannot continue:

- machine crashed
- script execution failed repeatedly
- API key/credits failed
- report card invalid
- promotion decision needs user judgment
- unexpected damage to protected anchors

Hermes watches for sentinel files and pings the user on Discord.
