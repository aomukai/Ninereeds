## Start the full generation run

**API key** — already in `.env`, script reads it automatically.
**Platform** — on Windows, use the direct runner. Do NOT use opencode.

```
python3 meta/scripts/story_gen_v2.py run --workers 4 2>&1 | tee tmp/story_gen_run.log
```

The run loops automatically through all modes until done:
`anchor → organic → floor_mop_up → standard_soft_mop_up → done`

**Safe to interrupt** (Ctrl+C) and resume at any time — state and tracker persist.
On restart the script syncs from tracker and picks up exactly where it stopped.

If API credits run out mid-run: recharge, then run the same command again with `-a`:
```
python3 meta/scripts/story_gen_v2.py run --workers 4 2>&1 | tee -a tmp/story_gen_run.log
```

## What the generator produces

Each story file in `training_data/teaching_stories/`:
- 4 language blocks: EN → DE → JP → ZH
- 1 `[user]`/`[Ninereeds]` pair per language (tiers 1–3), or 2 pairs (tier 4)
- Omniscient narrator, village world, no named characters
- JP: plain form only (never です/ます)
- ZH: Traditional Chinese only (never Simplified)
