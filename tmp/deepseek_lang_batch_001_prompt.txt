Task:
Read training_data/allowlist.txt.

Create simple multilingual "X is Y" files.

Output directory:
training_data/lang/lang_1/

Batch rule:
- Work from the top of training_data/allowlist.txt in order.
- Create files for the first 10 suitable English noun/concept entries only.
- Stop immediately after 10 files have been created.
- For entries examined before that point but skipped, keep a short reason.

For each suitable English noun/concept X, create:
training_data/lang/lang_1/X.md

where X is the English subject noun used in line 1.

File format:
Line 1: English sentence
Line 2: natural/localized German
Line 3: natural/localized Japanese
Line 4: natural/localized Chinese

Example:
training_data/lang/lang_1/dog.md

A dog is an animal.
Ein Hund ist ein Tier.
犬は動物だ。
狗是动物。

Rules:
- Use only content words that appear in training_data/allowlist.txt.
- Grammar/function words are allowed even if they are not in allowlist.txt.
- English and German must use correct articles where required.
- Keep sentences extremely simple.
- Use only classification/equivalence sentences of the form "X is Y".
- Do not use actions, locations, explanations, descriptions, or extra context.
- Do not create files for words that cannot naturally fit the pattern.
- Avoid abstract or philosophically ambiguous classifications.
- Do not add headings, comments, metadata, markdown, or blank lines inside the output files.
- Each file must contain exactly 4 lines.
- Use lowercase filenames based on the English X word.
- Replace spaces in filenames with underscores.
- Avoid duplicates.

Requirements:
- Create the directory if needed.
- Overwrite nothing outside training_data/lang/lang_1/.
- After writing the files, verify each created file has exactly 4 lines.

When finished, print only this plain-text summary to stdout:
created: <number>
output_dir: training_data/lang/lang_1/
skipped:
- <word>: <short reason>
