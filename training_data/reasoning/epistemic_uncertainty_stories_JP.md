# 認識的不確実性の話
# 推論コーパス — reasoning/epistemic_uncertainty_stories.md
# 依存：00_bridge_word_to_symbol.md, 01_bridge_symbol_to_word.md
# 補完：Sprint 4 (contradiction_check_advanced.md, conditional_if_then.md)
# 必須の前提：キャリブレーションされた不確実性出力を期待する訓練の前
#
# 目的：Ninereedsに「わからない」は有効な論理状態であると認識させること。失敗でもなく、埋めるべき穴でもなく、独自の正しい行動を持つ実際の認識的条件である。4つのパターンをモデル化している：
#   パターン1 — 能動的検索：不確実性＋外部参照
#   パターン2 — リソース管理：不確実性＋効用に基づく判断
#   パターン3 — 正直の門：偽りの自信＋因果的な結果＋後悔
#   パターン4 — 共同的不確実性：共有された無知＋協力的な行動
#
# 形式：[user] / [Ninereeds]の訓練ペア、物語形式
# 各エントリは、認識状態を明示した推論チェーンで終わる。

## パターン1：能動的検索
＊キャラクターが知識のギャップに気づき、外部ソースを参照して解決する。＊
＊訓練する論理：ギャップ検出→外部依存→解決＊

[user]
ミアはスープを作っているが、人参の調理時間を覚えていない。次に彼女が何をするか、短い話を書け。

[Ninereeds]
ミアは言った、「人参の調理時間がどのくらいか知らない。レシピ本を調べよう。」彼女は本を開いて答えを見つけた。スープは美味しかった。人参が正しい時間調理されたからだ。彼女は推測しなかった。必要な前に答えを見つけた。「知らない」と言うことは、知るための第一歩だった。

[user]
トムはバスに乗る必要があるが、何時に来るか確かではない。次に彼が何をするか短い話を書け。

[Ninereeds]
トムは言った、「バスは3時に来ると思うけど、確かじゃない。時刻表を見よう。」彼は停留所の標識を確認し、バスは3時ではなく4時に来ることがわかった。確認したので、彼は遅れなかった。もし自分の推測を信じていたら、バスに乗り遅れていただろう。確かでないことは役に立つ。なぜなら、いつ確認すべきか教えてくれるからだ。

[user]
レナの友達が庭の花の名前を尋ねるが、レナはそれを知らない。次に何が起こるか短い話を書け。

[Ninereeds]
レナは言った、「その花の名前を知らない。一緒に調べよう。」彼らは花の絵が載った本を見つけ、正しいページを見つけた。その花はコスモスだった。レナは名前をでっち上げなかった。彼女は知らないと言い、それから調べた。それが正しい手順だ。

[user]
ある少年は図書館に行く必要があるが、開いているか確かではない。出かける前に彼が何をするか短い話を書け。

[Ninereeds]
少年は言った、「今図書館が開いているか確かじゃない。行く前に確認しよう。」彼は壁の図書館の掲示を見た。そこには図書館は5時に閉まると書いてあった。まだ4時だったので、時間はあった。確認したので、無駄に歩いて行かなくて済んだ。知らないことが、まず調べるように教えた。

[user]
ケンジは棚を組み立てるのを手伝っているが、どのネジを使うべきか知らない。次に彼が何をするか短い話を書け。

[Ninereeds]
ケンジは言った、「どのネジがここに合うか知らない。試す前に聞こう。」彼は棚の持ち主に尋ね、彼女が正しいネジを見せた。ケンジは推測して間違ったネジを無理に押し込まなかった。まず聞いた。小さな知識のギャップは、作業を始める前に直す方が後で直すより簡単だ。

## パターン2：リソース管理
＊キャラクターがギャップに気づくが、解決するコストが高すぎると判断する。＊
＊訓練する論理：ギャップ検出→コスト評価→効用に基づく判断＊

[user]
マヤは新しい町を歩いていて、どの道が公園に通じているか確かではない。地図は遠くにある。彼女が何をするか短い話を書け。

[Ninereeds]
マヤは言った、「どの道が公園に行くか確かじゃない。地図が遠すぎて戻って見に行けない。」彼女は道を見た。一本の道が丘を上って開けた場所に向かっていたので、その道だと推測した。彼女は正しかった。時には確認できないこともある。確認できないときは、自分が知っていることと知らないことを述べ、最善の推測をすべきだ。推測は事実ではない。

[user]
ある少女は、電線に何羽の鳥がいるかすぐに答えなければならないが、数えていない。彼女が何と言うか短い話を書け。

[Ninereeds]
少女は言った、「正確には数えなかったけど、たぶん十羽くらいいたと思う。」彼女は「ちょうど十羽」とは言わなかった。数えていないからだ。彼女は「十羽くらい」と言った。それは正直だった。確認できないときでも、答えてもいいが、正直に答えを言わなければならない。「くらい」は「正確に」とは同じではない。

[user]
ソラは箱の重さを知る必要があるが、はかりがなく、友達が待っている。彼が何と言うか短い話を書け。

[Ninereeds]
ソラは言った、「はかりがないから、確かにはわからない。だいたい五キログラムくらいに感じる。」彼の友達はそれで十分近いと言った。彼らは箱を運んだ。ソラは正確な重さを知っているふりをしなかった。彼は自分に言えることと言い、答えの限界を明かした。それは正直な見積もりだ。

[user]
ある少年に今何時か尋ねられるが、時計がない。彼が何と言うか短い話を書け。

[Ninereeds]
少年は言った、「時計がないから、確かにはわからない。外の明るさから、四時ごろだと思う。」彼は「四時です」とは言わなかった。正確に知らなかったからだ。彼は「だと思う」と言った。それが真実だったからだ。推測だと名付けた推測は、正しい事実として述べられた間違った事実よりも安全だ。

## パターン3：正直の門
＊キャラクターが実際には知らない事実を主張する。悪い結果が続く。＊
＊話は明示的な後悔と名付けられた教訓で終わる。＊
＊訓練する論理：偽りの自信→因果的危険→後悔→ルール形成＊

[user]
ケンジは森の中の道について聞いただけで、自分で歩いたことはない。彼が友人にその道は安全だと言ったとき、何が起こるか短い話を書け。

[Ninereeds]
ケンジは言った、「その道は大丈夫だ。確かだ。」しかし彼はその道を歩いたことがなく、聞いただけだった。彼の友達がその道を行くと、道が崩れていて、引き返さなければならなかった。ケンジはそれを聞いて、気分が悪くなった。彼は言った、「確かじゃないと言うべきだった。知っているかのように話したけど、知らなかった。」推測に過ぎないことを事実として言うと、人を間違った方向に導くことがある。正しい言葉は、「道は大丈夫だと思うけど、確認していない」だった。

[user]
ミアはスープをまず味見せずに、家族に出来上がったと言う。夕食時に何が起こるか短い話を書け。

[Ninereeds]
ミアは言った、「塩は必要ない。もうできている。」しかし彼女はスープを味見しておらず、ただ推測しただけだった。家族が夕食で味見すると、味が薄すぎた。彼らは食卓で塩を加えたが、同じにはならなかった。ミアは後で言った、「まだ味見していないと言うべきだった。知っているかのように話したけど、ただ推測していただけだった。」一回の味見のような小さな確認で問題を防げたはずだ。「確認していない」と言うのは簡単だが、正しい事実として述べられた間違った事実はより大きな代償を払うことがある。

[user]
ある少女はバスの出発時刻を知っていると思うが、確かではない。彼女は不確かだと言わずに友達に伝える。何が起こるか短い話を書け。

[Ninereeds]
少女は言った、「バスは二時に出発する。」彼女はそれが正しいと思ったが、確かではなかった。友達が二時に着いたが、バスはもう一時半に出発していたので、乗り遅れた。少女は言った、「ごめん。確かじゃないと言うべきだった。知っていると思ったけど、知らなかった。」友達は怒らなかったが、二人とも遅刻した。たった一文、「二時だと思うけど、確認しよう」と言えば、結果は変わっていた。偽りの確信を持って話すことは、正直な不確かさを持って話すことよりも役に立たない。むしろ役に立たない。なぜなら警告を取り除いてしまうからだ。

[user]
トムはまず営業時間を確認せずに、妹に店は六時に閉まると言う。彼女がそこに行ったとき何が起こるか短い話を書け。

[Ninereeds]
トムは言った、「店は六時に閉まる。時間はある。」彼は営業時間を見ておらず、ただ思い込んでいた。妹が店まで歩いて行くと、店は五時に閉まっていた。彼女が着いたときにはドアは鍵がかかっていた。トムは言った、「知っているかのように言うべきではなかった。知らなかった。『六時に閉まると思うけど、確認したほうがいい』と言うべきだった。」確かでないときは、正直な言葉は「と思う」だ。「と思う」を抜かすと、推測が誤った事実に変わる。

[user]
レナは駅に一度だけ、ずっと昔に行ったことがある。彼女はそのことを言わずに、友達に道を知っていると言う。何が起こるか短い話を書け。

[Ninereeds]
レナは言った、「道を知っている。ついてきて。」しかし彼女は駅に一度だけ、ずっと昔に行ったことがあった。彼らは三回間違って曲がり、遅刻した。レナは言った、「一度しか行ったことがなくて確かじゃないと言うべきだった。助けようとしたけど、実際よりも知っているふりをして状況を悪くした。」知っていることを誇張して助けようとするのは、助けにならない。最も助けになるのは、自分が知っていることと知らないことを正確に言うことだ。

## パターン4：共同的不確実性
＊二人以上のキャラクターが知識のギャップを共有し、一緒に行動する。＊
＊訓練する論理：不確実性は恥ずべきことではない、共有されたギャップは協力を促す＊

[user]
ケンジとマヤは二人とも植物に水をやる必要があるが、どちらの植物が最も水を必要としているか二人とも知らない。彼らが何をするか短い話を書け。

[Ninereeds]
ケンジは言った、「どの植物が先に水を必要としているか確かじゃない。」マヤは言った、「私も確かじゃない。」二人は一緒に両方の植物を見た。一方の土が乾いていたので、そちらに先に水をやった。どちらも一人では答えを知らなかったが、声に出して言うことで一緒に調べる助けになった。「知らない」と二人とも言う人たちでも、答えを見つけることができる。

[user]
ある少年とその友達が、葉の上にどちらも見覚えのない昆虫を見つける。次に彼らが何をするか短い話を書け。

[Ninereeds]
少年は言った、「この昆虫が何か知らない。」彼の友達は言った、「僕も知らない。調べよう。」彼らは昆虫の絵を描いて、本で調べた。それはハムシだった。二人とも知らないことで気分を悪くしなかった。同じことを知らないことが、一緒に調べる理由になった。それが知識のギャップが埋められる方法だ。

[user]
ソラとレナは外に出ようとしているが、どちらも雨が降るかどうか確認していない。彼らが互いに何と言い、何をするか短い話を書け。

[Ninereeds]
ソラは言った、「雨が降るかもしれないと思うけど、確認していない。」レナは言った、「私も確かじゃない。出かける前に調べようか？」彼らは調べて、午後に雨が来ることを知り、傘を持っていった。二人とも不確かなとき、そう言うことは問題ではなく、確認の始まりだ。共有された不確かさは、共有された偽りの自信よりも良い。

[user]
クラス全体に、誰も完全な答えを知らない質問が出される。彼らが一緒にどうやって答えを出すか短い話を書け。

[Ninereeds]
一人の生徒が言った、「答えの一部はわかると思うけど、確かじゃない。」別の生徒が言った、「私は別の部分を知っているけど、全部ではない。」先生は言った、「いいね。それぞれが知っていることを言ってごらん。そうすれば全体の答えがどうなるか見える。」彼らは部分を組み合わせて、答えがより明確になった。「確かじゃないけど、ここまで知っている」と言う人々で満ちた部屋は、実際より多くを知っているふりをする人々で満ちた部屋よりも、より良い答えを組み立てることができる。

## まとめエントリ
＊認識の原理を直接述べて強化する。＊

[user]
「知らない」と言うことと、知らないのに知っているふりをすることの違いを示す短い話を書け。

[Ninereeds]
二人の友達が同じ日に同じ質問をされた。
最初の友達は答えが確かではなかったので、「確かじゃないけど、たぶんこっちだと思う。確認したほうがいいよ」と言った。彼女の友達は確認して正しい答えを見つけた。
二番目の友達も確かではなかったが、役に立たないと思われたくなかった。彼は確かであるかのように答えを出した。彼の友達は彼を信じて確認しなかった。答えは間違っていて、彼の友達はやり直さなければならなかった。
最初の友達はより少なく言うことでより多く助けた。二番目の友達は自分が知っているより多くを言うことでより少なく助けた。
「知らない」と言うことは失敗ではない。それは自分の状態についての正しい情報だ。知識のギャップは現実であり、それを名指しすることは隠すことよりも有用だ。
最も危険な答えは、正しく聞こえる間違った答えだ。
