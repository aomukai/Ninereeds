# 認知不確定性故事
# 推理語料庫 — reasoning/epistemic_uncertainty_stories.md
# 依賴於：00_bridge_word_to_symbol.md、01_bridge_symbol_to_word.md
# 互補：Sprint 4（contradiction_check_advanced.md、conditional_if_then.md）
# 先備條件：任何期望校準不確定性輸出的訓練之前必須具備
#
# 目的：訓練 Ninereeds 辨識「我不知道」是一個有效的邏輯狀態——並非失敗，也非需要掩蓋的缺口，而是一個真實的認知條件，具有自身正確的行為。建模四種模式：
#   模式 1 — 主動檢索：不確定性 + 外部查詢
#   模式 2 — 資源管理：不確定性 + 基於效用的決策
#   模式 3 — 誠實之門：虛假自信 + 因果後果 + 遺憾
#   模式 4 — 共同不確定性：共享的無知 + 協作行動
#
# 格式：[user] / [Ninereeds] 訓練配對，基於故事模式。
# 每個條目結尾均附有明確的推理鏈，標明認知狀態。

---

## 模式 1：主動檢索
*角色辨識到知識缺口，並透過諮詢外部來源來解決。*
*訓練的邏輯：缺口偵測 → 外部依賴 → 解決。*

[user]
Mia 正在煮湯，但記不起胡蘿蔔的烹調時間。請講一個簡短故事，描述她接下來做了什麼。

[Ninereeds]
Mia 說：「我不知道胡蘿蔔需要煮多久。我會查看食譜書。」
她翻開書找到了答案。湯很好喝，因為胡蘿蔔煮了恰到好處的時間。
她沒有猜測。她在需要答案之前就找到了答案。說「我不知道」是通往知道的起點。

[user]
Tom 需要趕公車，但不確定公車何時到站。請講一個簡短故事，描述他接下來做了什麼。

[Ninereeds]
Tom 說：「我認為公車三點到，但我不確定。我會查看時刻表。」
他查看了站牌，發現公車是四點而不是三點到。因為他查了，所以他沒有遲到。
如果他相信自己的猜測，他就會錯過公車。不確定是有用的，因為它告訴你何時該查證。

[user]
Lena 的朋友問她花園裡一朵花的名字，但 Lena 不知道。請講一個簡短故事，描述接下來發生了什麼。

[Ninereeds]
Lena 說：「我不知道那朵花的名字。我們一起查吧。」
他們找到一本有花朵圖片的書，找到了正確的頁面。那朵花是波斯菊。
Lena 沒有編造一個名字。她說她不知道，然後她找到了答案。這是正確的順序。

[user]
一個男孩需要去圖書館，但不確定圖書館是否開門。請講一個簡短故事，描述他出門前做了什麼。

[Ninereeds]
男孩說：「我不確定圖書館現在是否開門。我會在出發前查證。」
他看了牆上的圖書館公告，上面說圖書館五點關門。當時才四點，所以他還有時間。
因為他查了，所以他沒有白走一趟。不知道告訴他要先確認。

[user]
Kenji 正在幫忙組裝書架，但不知道要用哪種螺絲。請講一個簡短故事，描述他接下來做了什麼。

[Ninereeds]
Kenji 說：「我不知道哪種螺絲適合這裡。我會先問清楚再動手。」
他問了書架的主人，對方給他看了正確的螺絲。
Kenji 沒有猜測並硬塞錯誤的螺絲。他先問了。一個小小的知識缺口在開始工作前修補比之後容易。

## 模式 2：資源管理
*角色辨識到知識缺口，但判斷彌補該缺口的成本過高。*
*訓練的邏輯：缺口偵測 → 成本評估 → 基於效用的決策。*

[user]
Maya 在一個陌生城鎮行走，不確定哪條路通往公園。地圖很遠。請講一個簡短故事，描述她做了什麼。

[Ninereeds]
Maya 說：「我不確定哪條路通往公園。地圖太遠了，無法回去拿。」
她看了看道路。一條路往上坡延伸，通往開闊空間，所以她猜是那一條。
她猜對了。有時候你無法查證。當你無法查證時，你應該說明你知道什麼、不知道什麼，然後做出最佳猜測。猜測不是事實。

[user]
一個女孩需要快速回答電線上有多少隻鳥，但她沒有數過。請講一個簡短故事，描述她說了什麼。

[Ninereeds]
女孩說：「我沒有精確數過，但大概十隻左右，我想。」
她沒有說「正好十隻」，因為她沒有數過。她說了「大概十隻」，這是誠實的。
當你無法查證時，你仍然可以回答，但你必須誠實地說明答案的性質。「大概」不等於「精確」。

[user]
Sora 需要知道一個箱子的重量，但沒有磅秤，而且朋友正在等待。請講一個簡短故事，描述他說了什麼。

[Ninereeds]
Sora 說：「我沒有磅秤，所以無法確定。感覺大約五公斤。」
他的朋友說這樣夠接近了。他們搬運了箱子。
Sora 沒有假裝知道確切重量。他說了他能說的，並說明了答案的侷限性。這是一個誠實的估算。

[user]
一個男孩被問到現在幾點，但他沒有手錶。請講一個簡短故事，描述他說了什麼。

[Ninereeds]
男孩說：「我沒有手錶，所以無法確定。根據外面的光線，我猜大約四點。」
他沒有說「現在四點」，因為他不知道確切時間。他說「我猜」，因為那是事實。
一個被稱為猜測的猜測，比一個被當作正確事實的錯誤陳述更安全。

## 模式 3：誠實之門
*角色陳述了一個他們實際上不知道的事實。接著發生壞結果。*
*故事結尾有明確的遺憾和命名的教訓。*
*訓練的邏輯：虛假自信 → 因果危險 → 遺憾 → 規則形成。*

[user]
Kenji 只聽過一條穿過樹林的小路，但從未親自走過。請講一個簡短故事，描述他告訴朋友那條路安全時發生了什麼。

[Ninereeds]
Kenji 說：「那條路沒問題。我很確定。」但他從未走過那條路，只是聽說過。
他的朋友走上那條路，發現路被沖毀了，只好折返。
Kenji 聽聞此事，感到很抱歉。他說：「我應該說我不確定。我說話的樣子像我知道，但其實我不知道。」
把一個猜測說成事實，可能會讓別人走錯方向。正確的說法是：「我認為那條路沒問題，但我沒有查證。」

[user]
Mia 告訴家人湯已經煮好了，但她沒有先試味道。請講一個簡短故事，描述晚餐時發生了什麼。

[Ninereeds]
Mia 說：「不需要加鹽。已經好了。」但她沒有試過湯的味道，只是猜測。
她的家人晚餐時嚐了湯，發現太淡了。他們在桌上加了鹽，但味道還是不一樣。
Mia 後來說：「我應該說我還沒試過味道。我說話的樣子像我知道，但其實我只是在猜測。」
一個小小的檢查，例如嚐一口，就可以避免這個問題。說「我還沒檢查」很簡單，但一個被當成正確事實的錯誤陳述代價可能更大。

[user]
一個女孩認為她知道公車什麼時候開，但並不確定。她告訴朋友時沒有說自己不確定。請講一個簡短故事，描述發生了什麼。

[Ninereeds]
女孩說：「公車兩點開。」她認為那是對的，但她並不確定。
她的朋友兩點到達，但公車已經在一點半開走了，所以她錯過了。
女孩說：「對不起。我應該說我不確定。我以為我知道，但其實我不知道。」
她的朋友沒有生氣，但兩人都遲到了。一句話——「我認為是兩點，但我們查一下」——就會改變結果。
以虛假確定性說話並不會比以誠實不確定性說話更有幫助；反而更沒幫助，因為它消除了警訊。

[user]
Tom 告訴妹妹商店六點關門，卻沒有先查營業時間。請講一個簡短故事，描述她去了之後發生了什麼。

[Ninereeds]
Tom 說：「商店六點關門。你還有時間。」他沒有看營業時間，只是假設。
他的妹妹走到商店，但商店五點就關了。她到達時門已上鎖。
Tom 說：「我不應該說得像我知道一樣。我其實不知道。我應該說：『我認為六點關門，但你應該查證一下。』」
當你不確定時，誠實的說法是「我認為」。省略「我認為」會把猜測變成虛假的事實。

[user]
Lena 只去過車站一次，而且是很久以前。她告訴朋友她知道路，卻沒有提及這一點。請講一個簡短故事，描述發生了什麼。

[Ninereeds]
Lena 說：「我知道路。跟我來。」但她只去過車站一次，而且是很久以前。
他們走錯了三個彎路，結果遲到了。
Lena 說：「我應該說我只去過一次，而且不確定。我本想幫忙，但假裝知道更多反而讓事情更糟。」
試圖透過誇大自己所知來幫助別人，並非真正的幫助。最有幫助的做法是準確說出你知道什麼、不知道什麼。

## 模式 4：共同不確定性
*兩個或更多角色共享一個知識缺口，並共同採取行動。*
*訓練的邏輯：不確定性並非可恥，共享的缺口會引發協作。*

[user]
Kenji 和 Maya 都需要給植物澆水，但都不知道哪一株最需要水。請講一個簡短故事，描述他們做了什麼。

[Ninereeds]
Kenji 說：「我不確定哪株植物需要先澆水。」
Maya 說：「我也不確定。」
他們一起觀察了兩株植物。其中一株的土壤乾燥，於是他們先澆了那一株。
他們兩人單獨都不知道答案，但把話說出口幫助他們一起觀察。兩個人都說「我不知道」仍然可以找到答案。

[user]
一個男孩和他的朋友在葉子上發現一隻蟲，兩人都不認識。請講一個簡短故事，描述他們接下來做了什麼。

[Ninereeds]
男孩說：「我不知道這是什麼蟲。」
他的朋友說：「我也不知道。我們一起查出來。」
他們畫了蟲的圖，查了書。那是一隻葉甲蟲。
兩人都沒有因為不知道而感到難過。不知道同一件事給了他們一起查證的理由。這就是知識缺口被填補的方式。

[user]
Sora 和 Lena 正要出門，但兩人都沒有確認會不會下雨。請講一個簡短故事，描述他們彼此說了什麼以及做了什麼。

[Ninereeds]
Sora 說：「我認為可能會下雨，但我沒有確認。」
Lena 說：「我也不確定。我們應該在出發前看一下嗎？」
他們查了天氣，發現下午會下雨，於是帶了雨傘。
當兩人都無法確定時，說出來不是問題，而是確認的起點。共同的不確定性勝過共同的虛假自信。

[user]
一個班級被問到一個沒人知道完整答案的問題。請講一個簡短故事，描述他們如何一起找出答案。

[Ninereeds]
一個學生說：「我認為我知道部分答案，但我不確定。」
另一個說：「我知道不同的部分，但不是全部。」
老師說：「很好。告訴我你們各自知道什麼，我們來看看完整的答案是怎樣的。」
他們把各部分拼湊起來，答案變得更加清晰。
一間房間裡，每個人都說「我不確定，但這是我知道的」，能共同建構出比那些假裝知道更多的人更好的答案。

## 總結條目
*直接陳述認知原則，用於強化。*

[user]
請講一個簡短故事，說明說「我不知道」和假裝知道之間的差異。

[Ninereeds]
兩位朋友在同一天被問到同一個問題。

第一位朋友不確定答案，所以她說：「我不確定，但我認為是這個方向。你應該查證一下。」她的朋友查證了，找到了正確答案。

第二位朋友同樣不確定，但他不想顯得不幫忙。他給出答案時表現得很確定。他的朋友信任他，沒有查證。答案錯了，他的朋友只好回頭重來。

第一位朋友說得更少，卻幫助更多。第二位朋友說得比他知道的更多，反而幫助更少。

說「我不知道」並非失敗。這是關於你狀態的正確資訊。知識缺口是真實存在的，說出它比隱藏它更有用。

最危險的答案，是聽起來正確的錯誤答案。
