# Wiki Level 2 Expansion Assessment

This file records the file-level assessment for whether a current Level 1 wiki file should stay at Level 1 fidelity or receive a dedicated Level 2 expansion article.

Use this together with:
- `history.md` — role groups and current file counts
- `training_data/wiki/level1_finish_and_level2_start_plan.md` — explicit Level 2 candidates and compact-file warnings
- `training_data/wiki/wiki_level2_queue.md` — filtered production queue for actual Level 2 expansion candidates

---

## Decision rules used

1. Files explicitly named under **Good Level 2 candidates** in `level1_finish_and_level2_start_plan.md` are the safest expand-now set.
2. Files explicitly named under **Files that should probably stay compact** should not get automatic Level 2 articles.
3. `World-anchor inventories` usually stay broad and passive unless a file also supports many child-facing scripts.
4. `Example-bearing pragmatic and social sets` are more likely to justify expansion when they naturally branch into common situations, subcases, and repair patterns.
5. `Bridge and modifier categories` usually stay compact unless repeated story/wiki pressure proves they need more depth.
