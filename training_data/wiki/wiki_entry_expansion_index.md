# Wiki Entry Expansion Index

This is the canonical **entry-level** index for wiki expansion decisions.

A wiki file is a container. Each entry inside the file is treated as its own entity for escalation decisions.
That means a Level 2 article may cover only a subset of a source file's Level 1 entries.
Entries not selected for deeper expansion remain at their lower current level and stay visible here.

Use this together with:
- `training_data/wiki/wiki_expansion_index.md` — file-level rollup
- `training_data/wiki/wiki_level2_queue.md` — file containers to process one at a time
- `training_data/wiki/level2_articles_manifest.md` — article draft status
- `training_data/wiki/wiki_entry_expansion_index.csv` — machine-friendly full index

## Index format

- `current_max_level` = highest written level currently detected for that entry
- `status=level1_only` means the entry has not yet been escalated
- `status=has_level2` means the entry is currently represented in a written Level 2 article
- Detection for already-written files is heuristic and may need manual correction during later passes
