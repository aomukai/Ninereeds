# Wiki Level 2 Queue

This file is the working queue for **dedicated Wiki Level 2 expansion articles**.

This queue is intentionally filtered. It does **not** include every Level 1 wiki file.
Only files that passed the Level 2 expansion assessment belong here.

Use this together with:
- `training_data/wiki/wiki_level2_expansion_assessment.md` — file-level expand/keep decisions
- `training_data/wiki/wiki_entry_expansion_index.md` — entry-level entity index and rollup
- `training_data/wiki/wiki_entry_expansion_index.csv` — machine-friendly entry-level index
- `training_data/wiki/02_wiki_implementation_todo.md` — active cross-phase tracker
- `training_data/wiki/01_CORPUS_STATUS.md` — corpus status and role groups
- `training_data/wiki/level1_finish_and_level2_start_plan.md` — cadence constraints and explicit candidate guidance

---

## Queue rules

- Work **one file container at a time**.
- Each source file is a container of Level 1 entries; **each entry inside it must be evaluated individually** for escalation.
- This queue covers only files approved for dedicated Level 2 expansion articles.
- A Level 2 article does **not** need to carry every Level 1 entry from the source file.
- Entries that do not earn escalation stay at their lower current level and must remain visible in `wiki_entry_expansion_index.md` / `.csv`.
- Finish the full **creation pass** before starting the quality pass.
- Finish the full **quality pass** before starting the gap pass.
- Finish the full **gap pass** before starting the dependency pass.
- Keep the same file order across all four passes unless a human explicitly reprioritizes it.
- Narrative summaries belong in `01_CORPUS_STATUS.md`, not here.
- The human review gate before actual Level 2 writing still applies.
