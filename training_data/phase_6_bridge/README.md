# Phase 6 Bridge

Working directory for the post-Phase-5 bridge curriculum.

Purpose:
- add the missing high-frequency scaffold vocabulary that Phase 1–5 does not currently teach explicitly
- teach those words inside a small reusable pattern grid rather than only as isolated definitions
- prepare the dragon for Story Layer 1 and later dialogue handling without collapsing too early into free-form quotation patterns

Use this directory together with:
- `training_data/phase 1 to 5/rewritten/missing_curriculum_terms.md`
- `training_data/phase_6_bridge/phase_6_bridge_spec.md`
- `training_data/phase_6_bridge/phase_6_bridge_manifest.md`
- `training_data/phase_6_bridge/story_dialogue_progression.md`
- `training_data/wiki/story_layer_rules.md`
- `docs/training_pipeline.md`

This directory is for:
- bridge-curriculum planning
- file manifests
- pattern-grid rules
- later Phase 6 curriculum files once implementation begins

This directory is not for:
- wiki-entry drafting
- story-layer corpora
- ad hoc experimental notes that should live elsewhere

## Intended role in the training sequence

Phase 6 Bridge sits between:
1. Phase 1–5 curriculum foundation
2. Wiki Level 1 grounding work
3. Story Layer 1

It should teach the abstract connective vocabulary that stories and later wiki layers rely on, while still staying controlled, repetitive, and curriculum-like.

## Initial implementation targets

The first weekend implementation pass should focus on:
- creating the Phase 6 bridge spec and manifest
- choosing the first bridge word families
- defining the minimal pattern grid per family
- drafting the first batch of actual Phase 6 files in the repo's standard curriculum style
- documenting the dialogue progression so story generation does not jump too early into raw quoted speech
