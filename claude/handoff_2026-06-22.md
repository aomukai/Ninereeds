2. **C16A aug pass** (`angle_aug.py --wave 1`) kept deadlocking due to two bugs:
   - `log()` and `append_to()` shared the same `_lock` — one blocked USB write held the lock forever

3. **Both bugs fixed** in `meta/scripts/angle_aug.py`:
   - `log()` no longer uses a lock — uses its own file (`tmp/aug_rephrase.log`) with a try/except so a log failure never kills a worker

## First thing to do in new session

**Kill any lingering workers from the USB path**, then start fresh from `~/Ninereeds`:

```bash
# 1. Kill any leftover workers (check first)
ps aux | grep angle_aug | grep -v grep

# 2. Clear claim files
find ~/Ninereeds/training_data/redesign/aug_claims -name "*.claim" -delete

# 3. Launch all three sources in parallel for ~3× speed
cd ~/Ninereeds
set -a && source .env && set +a

python3 meta/scripts/angle_aug.py gen --wave 1 --source openrouter --workers 4 > /dev/null 2>&1 &
python3 meta/scripts/angle_aug.py gen --wave 1 --source deepseek   --workers 4 > /dev/null 2>&1 &
python3 meta/scripts/angle_aug.py gen --wave 1 --source nvidia     --workers 1 > /dev/null 2>&1 &

bash meta/scripts/angle_aug_monitor.sh &
```

> Note: NVIDIA NIM is rate-limited — 1 worker only.
