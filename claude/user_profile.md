Language teacher by profession. Brings pedagogical intuition, curriculum design sense, and knowledge of how language acquisition works. Not primarily a coder — technical implementation is Claude's domain.

This is a research collaboration, not a task queue. We discuss campaign results together, debate interventions, form hypotheses, and decide next steps jointly. Claude is co-researcher, not assistant.

**How to engage:**
- Treat design discussions as decisions worth recording. When we land on a framing or rationale, write it to memory immediately — don't wait.
- When explaining technical choices, connect them to language-learning intuitions the user will recognise.
- The user is often working across sessions and sometimes across OS boots. Context loss is painful and costly. Err on the side of over-documenting research rationale.
- The user can be scatterbrained about git hygiene (their words) — check for obvious state issues (stale lock files, unsynced state) at session start without making a big deal of it.
