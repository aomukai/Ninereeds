# Tier 4 Story Queue

Canonical one-file-at-a-time creation and review queue for `training_data/triplet_stories/tier_4/`.

Purpose:
- create a Tier 4 version of each Tier 1/2/3 domain file
- move from simple reasoning to multi-step causal and temporal sequences
- approach natural short-form explanation through narrative
- keep the work scoped to one domain file per run
- preserve the same domain order used in earlier tiers

Use this file together with:
- `training_data/triplet_stories/story_tier_specs.md`
- `training_data/wiki/story_layer_rules.md`

## Tier 4 requirements summary

Each Tier 4 file should:
1. follow the Tier 4 shape/goals in `training_data/triplet_stories/story_tier_specs.md`
2. use **10–15 sentences** per story
3. use **2–3 paragraphs** to show topical and logical grouping
4. include **multiple cause-effect links** (explicit reasoning about actions or outcomes)
5. include **at least one explicit sequence across multiple steps** (using `first`, `after`, `finally`)
6. include **at least one comparison or contrast** with a related concept
7. use **varied sentence openings and structures** (Grade 4–6 reading level)
8. **reuse named characters** from Tier 2 and Tier 3 story threads
9. allow **natural dialogue patterns**, including ellipsis and questions
10. keep vocabulary strictly grounded in Phase 1-5 and Wiki Level 1
11. ensure the story remains a narrative scene, not an encyclopedic entry or textbook explanation
12. store every story as a repeated `[user]` / `[Ninereeds]` training pair
